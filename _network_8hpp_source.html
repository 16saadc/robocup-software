00003 // Here's how networking works:
00004 //
00005 // Vision:  Shared or simulated.
00006 //    Shared vision is the system used in competition.  It sends multicast data to 224.5.20.2 port 10002.
00007 //    Simulated vision is sent by our simulator to 127.0.0.1 ports 10000 and 10001.
00008 //      The data sent to the two simulated vision ports is identical
00009 //      It's sent to two ports so we can stay on localhost since multicast
00010 //      doesn't work if there are no routing entries (a common case during development).
00011 //
00012 // Radio:
00013 //    There are two radio channels from the network's point of view.
00014 //    Currently the hardware only supports one, but this needs to be fixed.
00015 //    Radio packets stay on localhost.
00016 //
00017 //    Soccer uses ports 12000 and 12001.
00018 //    Radio uses ports 13000 and 13001.
00019 //
00020 //    RadioTx packets go from soccer to radio.
00021 //    RadioRx packets go from radio to soccer.
00022 //
00023 // The network ports are set in Processor's constructor and don't change after that.
00024 // They are determined by command-line options (-sim and -r).
00025 // If no radio channel is given on the command line, the first available one is picked
00026 // based on which soccer-side port can be bound.
